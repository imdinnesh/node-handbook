# Command Line Output & the Console Module

The Node.js `console` module provides several methods for interacting with the command line. It is highly similar to the `console` object found in modern web browsers but optimized for terminal environments.

---

## 1. Basic Logging

The most common method is `console.log()`, which prints a string (or an object converted to a string) to the standard output.

```javascript
const x = 'x';
const y = 'y';
console.log(x, y); // Prints both variables separated by a space
```

### Format Specifiers
You can create formatted strings by passing variables along with specific escape sequences.

| Specifier | Description |
| :--- | :--- |
| `%s` | Format as a **String**. |
| `%d` | Format as a **Number**. |
| `%i` | Format as an **Integer**. |
| `%o` | Format as an **Object**. |

**Example:**
```javascript
console.log('My %s has %d ears', 'cat', 2);
console.log('%o', Number);
```

---

## 2. Utility Methods

### Clearing the Console
`console.clear()` clears the terminal screen. The exact behavior may vary depending on the terminal emulator used.

### Counting Elements
`console.count()` keeps a running tally of how many times a specific label has been logged.

```javascript
console.count('orange'); // orange: 1
console.count('orange'); // orange: 2
console.count('apple');  // apple: 1

// Reset the counter
console.countReset('orange');
console.count('orange'); // orange: 1
```

### Stack Tracing
`console.trace()` is useful for debugging as it prints the full call stack to the current execution point.

```javascript
const function2 = () => console.trace();
const function1 = () => function2();
function1();
```

### Performance Timing
Measure the execution time of code blocks using `console.time()` and `console.timeEnd()`.

```javascript
console.time('myTimer');

// Run some logic...
console.log('Task complete');

console.timeEnd('myTimer'); // Prints: myTimer: [time]ms
```

---

## 3. Standard Streams: stdout vs. stderr

Behind the scenes, the console module writes to different system streams:

*   **`console.log` / `console.info`**: Writes to **stdout** (Standard Output).
*   **`console.error` / `console.warn`**: Writes to **stderr** (Standard Error).

> [!TIP]
> This distinction is useful because you can redirect errors to a separate log file while keeping the normal output in the terminal.

## 4. Advanced Terminal Styling (`styleText`)

> [!IMPORTANT]
> **Active Development**: The `styleText` utility was introduced in v20.12.0/v21.7.0 and is stable as of Node.js v22.11.0.

You can style your terminal output using the `styleText` function from the `node:util` module without external dependencies like `chalk`.

```javascript
import { styleText } from 'node:util';

console.log(
  styleText(['red'], 'This is red text ') +
  styleText(['green', 'bold'], 'and this is green bold text ') +
  'this is normal text'
);
```

The first argument is an array of style keys (e.g., `red`, `bold`, `underline`), and the second is the string to be styled.
