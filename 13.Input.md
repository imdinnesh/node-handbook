# Accepting Command Line Input in Node.js

To build interactive CLI programs, Node.js provides the built-in `readline` module. It allows you to read input from a readable stream, such as `process.stdin`, one line at a time.

---

## 1. Using the `readline` Module (Callback-based)

The standard way to capture input since Node.js v7 is by creating an interface and using the `question()` method.

```javascript
const readline = require('node:readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

rl.question("What's your name? ", (name) => {
  console.log(`Hi ${name}!`);
  
  // Always close the interface when done
  rl.close();
});
```

### How it Works:
1.  **`createInterface`**: Connects the module to `stdin` (input) and `stdout` (output).
2.  **`question()`**: Displays a prompt and waits for the user to type and press **Enter**.
3.  **Callback**: Executed once the input is received.
4.  **`rl.close()`**: Relinquishes control of the streams and allows the process to exit.

---

## 2. Modern Approach: `readline/promises`

Since Node.js v17, you can use the Promise-based API, which is much cleaner when combined with `async/await`.

```javascript
const readline = require('node:readline/promises');

async function askName() {
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
  });

  const name = await rl.question("What's your name? ");
  console.log(`Hi ${name}!`);

  rl.close();
}

askName();
```

## Security & Best Practices

> [!TIP]
> **Handling Passwords**: When asking for sensitive information like passwords, it is best practice not to echo the characters back to the terminal (or show `*` symbols instead) to prevent over-the-shoulder security risks.

### Summary
The `readline` module is powerful and offers many other methods for handling history, tab completion, and more. For complex CLI interactions, consider exploring the [official documentation](https://nodejs.org/api/readline.html).
