# Writing Files with Node.js

Node.js provided several ways to write data to files. Depending on your needs, you can choose between callback-based, synchronous, or promise-based APIs.

---

## 1. Basic File Writing

The primary method for writing to files is `fs.writeFile()`. By default, this API will **replace** the contents of the file if it already exists.

### Asynchronous (Callback)
This is the non-blocking way to write files using a callback function.

```javascript
const fs = require('node:fs');

const content = 'Some content!';

fs.writeFile('/Users/joe/test.txt', content, err => {
  if (err) {
    console.error(err);
    return;
  }
  // File written successfully
  console.log('File written successfully');
});
```

### Synchronous
If you need to block execution until the file is written, use the synchronous version.

```javascript
const fs = require('node:fs');

const content = 'Some content!';

try {
  fs.writeFileSync('/Users/joe/test.txt', content);
  console.log('File written successfully');
} catch (err) {
  console.error(err);
}
```

### Promise-based (Recommended)
Modern Node.js applications typically use the `fs/promises` module for better readability with `async/await`.

```javascript
const fs = require('node:fs/promises');

async function example() {
  try {
    const content = 'Some content!';
    await fs.writeFile('/Users/joe/test.txt', content);
    console.log('File written successfully');
  } catch (err) {
    console.error(err);
  }
}

example();
```

---

## 2. Using Write Flags

You can modify the default "overwrite" behavior by specifying a flag in the options object.

```javascript
fs.writeFile('/Users/joe/test.txt', content, { flag: 'a+' }, err => {});
```

### Common Flags Table

| Flag | Description | Creates file if missing? |
| :--- | :--- | :---: |
| `r+` | Opens the file for reading and writing. | ❌ |
| `w+` | Opens for reading/writing; positions stream at the **beginning**. | ✅ |
| `a`  | Opens for writing; positions stream at the **end**. | ✅ |
| `a+` | Opens for reading/writing; positions stream at the **end**. | ✅ |

> [!TIP]
> You can find more comprehensive information about all available flags in the official [Node.js FS documentation](https://nodejs.org/api/fs.html#file-system-flags).

---

## 3. Appending Content to a File

Appending is useful when you want to add data to the end of an existing file without losing its current content.

### Callback Version
```javascript
const fs = require('node:fs');

const content = '\nNew log entry!';

fs.appendFile('file.log', content, err => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Content appended successfully');
});
```

### Promises Version
```javascript
const fs = require('node:fs/promises');

async function example() {
  try {
    const content = '\nNew log entry!';
    await fs.appendFile('/Users/joe/test.txt', content);
    console.log('Content appended successfully');
  } catch (err) {
    console.error(err);
  }
}

example();
```

> [!IMPORTANT]
> When appending, consider adding newline characters (`\n`) if you want each entry to appear on a new line.
