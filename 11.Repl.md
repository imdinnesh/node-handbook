# Node.js REPL (Read-Eval-Print Loop)

The Node.js REPL is an interactive programming environment that takes user input, evaluates it, and returns the result to the console. It is an excellent tool for testing small snippets of JavaScript code without creating a file.

---

## 1. Basic Usage

To start a REPL session, simply run the `node` command in your terminal without any arguments:

```bash
node
```

Once started, the prompt changes to `>`, indicating it is waiting for JavaScript code.

### Evaluation vs. Expressions
Node automatically evaluates and prints the result of any line you enter.

```text
> console.log('test')
test
undefined

> 5 === '5'
false
```
*   **Note**: `console.log()` returns `undefined`, which is why you see it after the printed string.
*   **Expressions**: Directly entering an expression (like `5 === '5'`) returns its evaluation result immediately.

---

## 2. Interactive Features

### Multi-line Support
The REPL automatically detects when a statement is incomplete (e.g., after an open bracket `{`) and enters multi-line mode, indicated by `...`.

```text
> [1, 2, 3].forEach(num => {
... console.log(num)
... })
1
2
3
undefined
```

### Special Variables & Keys
*   **The `_` Variable**: Typing `_` after an operation returns the result of the last evaluated expression.
*   **Up Arrow**: Accesses the history of previously executed commands across current and past sessions.

---

## 3. Dot Commands

The REPL includes special internal commands that start with a dot (`.`).

| Command | Description |
| :--- | :--- |
| `.help` | Shows all available dot commands. |
| `.editor` | Enables editor mode for writing complex multi-line code easily. Press `Ctrl+D` to finish. |
| `.break` / `.clear` | Aborts a multi-line expression or resets the context. |
| `.load` | Loads a JavaScript file into the current session. |
| `.save` | Saves the current REPL session history to a file. |
| `.exit` | Exits the REPL (can also press `Ctrl+C` twice). |

---

## 4. Programmatic REPL

You can embed a custom REPL instance within your own Node.js applications using the `node:repl` module.

### Starting a Custom REPL
```javascript
const repl = require('node:repl');

// Start the REPL with a custom prompt
const local = repl.start({
  prompt: '$ ',
  useGlobal: true
});

// Handle the exit event
local.on('exit', () => {
  console.log('\nExiting REPL...');
  process.exit();
});
```

### Use Case
This is useful for creating custom interactive shells for debugging or managing internal application state in real-time.
