# Introduction to npm

**npm** is the standard package manager for Node.js.

In September 2022, over 2.1 million packages were reported being listed in the npm registry, making it the biggest single language code repository on Earth. You can be sure there is a package for (almost!) everything.

It started as a way to download and manage dependencies of Node.js packages, but it has since become a tool used extensively in frontend JavaScript.

> **Alternatives**: Yarn and pnpm are popular alternatives to the npm CLI.

---

## Packages

npm installs, updates, and manages downloads of **dependencies** of your project. Dependencies are pre-built pieces of code, such as libraries and packages, that your Node.js application needs to work.

---

## Installing Dependencies

### Installing all dependencies
If a project has a `package.json` file, running the following command will install everything the project needs in the `node_modules` folder:

```bash
npm install
```

### Installing a single package
You can install a specific package by running:

```bash
npm install <package-name>
```

Since npm 5, this command automatically adds the package to your `package.json` dependencies.

### Common Flags
- `--save-dev` (`-D`): Installs and adds the entry to `devDependencies`.
- `--no-save`: Installs but does not add the entry to `package.json`.
- `--save-optional` (`-O`): Installs and adds the entry to `optionalDependencies`.
- `--no-optional`: Prevents optional dependencies from being installed.

**Note on Dependencies**: 
- `devDependencies` are for development tools (like testing libraries).
- `dependencies` are bundled with the app in production.
- `optionalDependencies` won't cause installation to fail if they can't be built, but your code must handle their absence.

---

## Updating Packages

Updating is made easy by running:

```bash
npm update
```

npm will check all packages for a newer version that satisfies your **versioning constraints**. You can also update a single package:

```bash
npm update <package-name>
```

---

## Versioning

npm manages versioning using the **Semantic Versioning (semver)** standard. This allows you to specify specific versions or ranges, which is useful when:
- A library is only compatible with a specific major release of another.
- A bug in the latest release needs to be avoided.
- You want to keep the whole team on the exact same version.

To install a **specific version**, run:

```bash
npm install <package-name>@<version>
```

---

## Running Tasks

The `package.json` file supports specifying command-line tasks that can be run using:

```bash
npm run <task-name>
```

### Examples

**Basic Scripts:**
```json
{
  "scripts": {
    "start-dev": "node lib/server-development",
    "start": "node lib/server-production"
  }
}
```

**Webpack Scripts:**
```json
{
  "scripts": {
    "watch": "webpack --watch --progress --colors --config webpack.conf.js",
    "dev": "webpack --progress --colors --config webpack.conf.js",
    "prod": "NODE_ENV=production webpack -p --config webpack.conf.js"
  }
}
```

Instead of typing long commands, you can simply run:
```bash
npm run watch
npm run dev
npm run prod
```
