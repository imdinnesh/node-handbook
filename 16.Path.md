# Node.js File Paths

Every file in a system has a unique path. However, path structures differ across operating systems:

*   **Linux/macOS:** `/users/joe/file.txt`
*   **Windows:** `C:\users\joe\file.txt`

> [!IMPORTANT]
> Always use the built-in `path` module to handle file paths. It automatically accounts for these OS differences, ensuring your application remains cross-platform compatible.

To use this module, include it at the top of your file:
```javascript
const path = require('node:path');
```

---

## 1. Extracting Information from a Path

Given a file path, you can extract specific components using the following methods:

| Method | Description |
| :--- | :--- |
| `path.dirname()` | Retrieves the parent directory of a file. |
| `path.basename()` | Retrieves the filename part of the path. |
| `path.extname()` | Retrieves the file's extension. |

### Example
```javascript
const path = require('node:path');
const notes = '/users/joe/notes.txt';

console.log(path.dirname(notes));  // /users/joe
console.log(path.basename(notes)); // notes.txt
console.log(path.extname(notes));  // .txt
```

#### Removing the Extension
You can get the filename *without* the extension by passing a second argument to `basename`:
```javascript
path.basename(notes, path.extname(notes)); // notes
```

---

## 2. Working with Paths

### Joining Paths
Use `path.join()` to combine two or more path segments. This method correctly handles slashes based on the operating system.

```javascript
const name = 'joe';
path.join('/', 'users', name, 'notes.txt'); // '/users/joe/notes.txt'
```

### Resolving Absolute Paths
You can calculate the absolute path of a relative path using `path.resolve()`:

```javascript
path.resolve('joe.txt'); // '/Users/joe/joe.txt' (if run from home folder)
```

Node.js appends the relative segment to the **Current Working Directory (CWD)**. You can also specify a base folder:

```javascript
path.resolve('tmp', 'joe.txt'); // '/Users/joe/tmp/joe.txt'
```

If the first parameter starts with a slash, it is treated as an absolute path:
```javascript
path.resolve('/etc', 'joe.txt'); // '/etc/joe.txt'
```

---

## 3. Path Normalization

The `path.normalize()` function calculates the "true" path by resolving relative specifiers like `.` or `..` and correcting double slashes.

```javascript
path.normalize('/users/joe/..//test.txt'); // '/users/test.txt'
```

> [!WARNING]
> Neither `resolve()` nor `normalize()` check if the file or directory actually exists on the disk. They strictly perform string-based path calculations.
