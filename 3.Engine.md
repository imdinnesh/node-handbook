# The V8 JavaScript Engine

V8 is the name of the **JavaScript engine** that powers Google Chrome. It's the component that takes our JavaScript and executes it while browsing.

V8 is specifically the engine that parses and executes JavaScript code. The **DOM** and other **Web Platform APIs** (which make up the runtime environment) are provided by the browser.

---

## Browser Independence

The cool thing is that the JavaScript engine is **independent of the browser** in which it's hosted. This key feature enabled the rise of Node.js. V8 was chosen to power Node.js in 2009, and as popularity exploded, it became the engine that now powers an incredible amount of server-side code.

The Node.js ecosystem is huge, and thanks to V8, it also powers desktop apps with projects like **Electron**.

---

## Other JS Engines

Other browsers have their own JavaScript engine:

- **Firefox**: SpiderMonkey
- **Safari**: JavaScriptCore (also called Nitro)
- **Edge**: Originally based on Chakra, but recently rebuilt using Chromium and the **V8 engine**.

All those engines implement the **ECMA ES-262 standard**, also called **ECMAScript**, the standard used by JavaScript.

---

## The Quest for Performance

V8 is written in **C++** and is continuously improved. It is portable and runs on Mac, Windows, Linux, and several other systems. 

V8 is always evolving to speed up the Web and the Node.js ecosystem. On the web, there is a race for performance that's been going on for years, and we benefit from this competition through faster and more optimized machines year after year.

> *Note: In this introduction, we will ignore the specific implementation details of V8 as they change over time and can be found on more authoritative sites like the V8 official site.*

---

## Compilation vs. Interpretation

JavaScript is generally considered an interpreted language, but modern JavaScript engines no longer just interpret JavaScript—**they compile it**.

This shift began in 2009 with the addition of the SpiderMonkey compiler to Firefox 3.5. Today, JavaScript is internally compiled by V8 with **Just-In-Time (JIT) compilation** to speed up execution.

### Why Compile?
Since the introduction of Google Maps in 2004, JavaScript has evolved from simple scripts to complete applications with hundreds of thousands of lines of code. Our applications now run for hours inside a browser.

In this new world, compiling JavaScript makes perfect sense: while it might take slightly longer to prepare the code, the resulting execution is **much more performant** than purely interpreted code.
