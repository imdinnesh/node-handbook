# Node.js with WebAssembly (Wasm)

WebAssembly is a high-performance, assembly-like language that can be compiled from multiple source languages including C/C++, Rust, and AssemblyScript. It is a portable target for compilation, supported by all major browsers and **Node.js**.

The specification defines two file formats:
*   **Binary format (.wasm)**: A compact, efficient format for execution.
*   **Text format (.wat)**: A human-readable representation of the binary.

---

## Key Concepts

| Concept | Description |
| :--- | :--- |
| **Module** | A compiled binary file (the `.wasm` file). |
| **Memory** | A resizable `ArrayBuffer` that holds data for the module. |
| **Table** | A resizable typed array of references (e.g., functions) not stored in Memory. |
| **Instance** | A Module paired with its Memory, Table, and variables for execution. |

To interact with these, Node.js provides a global `WebAssembly` object:

```javascript
console.log(WebAssembly);

/* Output:
Object [WebAssembly] {
  compile: [Function: compile],
  validate: [Function: validate],
  instantiate: [Function: instantiate],
  ...
}
*/
```

## Generating WebAssembly Modules

There are several ways to generate `.wasm` binary files:

1.  **Hand-writing (.wat)**: Write the text format by hand and convert to binary using tools like [wabt](https://github.com/WebAssembly/wabt).
2.  **C/C++**: Use the [Emscripten](https://emscripten.org/) toolchain.
3.  **Rust**: Use [wasm-pack](https://rustwasm.github.io/wasm-pack/) for seamless integration.
4.  **AssemblyScript**: A TypeScript-like language specifically designed for WebAssembly.

> [!NOTE]
> Many of these tools generate the binary file along with JavaScript "glue" code to handle instantiation and browser/OS integration.

## How to Use WebAssembly in Node.js

Once you have a `.wasm` file, you can instantiate it using the `WebAssembly.instantiate` method.

### CommonJS (CJS) Example
Assuming you have an `add.wasm` file that exports a single `add` function:

```javascript
const fs = require('node:fs');

// 1. Read the .wasm file into a buffer
const wasmBuffer = fs.readFileSync('./path/to/add.wasm');

// 2. Instantiate the module
WebAssembly.instantiate(wasmBuffer).then(wasmModule => {
  // 3. Access exported functions from instance.exports
  const { add } = wasmModule.instance.exports;
  
  const sum = add(5, 6);
  console.log(sum); // Outputs: 11
});
```

## Interacting with the OS (WASI)

By design, WebAssembly modules are sandboxed and cannot directly access the operating system (disk, network, etc.). To bridge this gap, the **WebAssembly System Interface (WASI)** was created.

*   **Wasmtime**: A popular standalone runtime for WebAssembly that implements WASI.
*   **Node.js**: Also provides a `wasi` core module to allow WebAssembly modules to securely interact with the OS.
