# Working with File Descriptors in Node.js

Before you can interact with a file in your filesystem, you must first obtain a **File Descriptor**.

A file descriptor is a unique identifier (a number, `fd`) returned by the operating system when a file is opened. It acts as a reference to an open file, allowing you to perform various operations like reading, writing, or closing.

---

## Opening a File (Asynchronous)

The most common way to get a file descriptor is using the `fs.open()` method from the `fs` module:

```javascript
const fs = require('node:fs');

fs.open('/Users/joe/test.txt', 'r', (err, fd) => {
  if (err) {
    console.error('Error opening file:', err);
    return;
  }
  // fd is our file descriptor
  console.log('File descriptor obtained:', fd);
});
```

### Common Flags

The second parameter (`'r'` in the example above) is the **flag**. It specifies the mode in which the file should be opened.

| Flag | Description | Auto-Create File? |
| :--- | :--- | :---: |
| `r` | Open for reading (default). | ❌ |
| `r+` | Open for reading and writing. | ❌ |
| `w+` | Open for reading and writing; resets pointer to start. | ✅ |
| `a` | Open for writing; positions pointer at end (**Append**). | ✅ |
| `a+` | Open for reading and writing; positions pointer at end. | ✅ |

---

## Synchronous Opening

You can also open a file synchronously using `fs.openSync`. This returns the file descriptor directly rather than using a callback.

```javascript
const fs = require('node:fs');

try {
  const fd = fs.openSync('/Users/joe/test.txt', 'r');
  console.log('Synchronous FD:', fd);
} catch (err) {
  console.error('An error occurred:', err);
}
```

> [!NOTE]
> Synchronous methods block the event loop until the operation completes. Use them sparingly in production servers.

---

## Promise-based Approach (`fs/promises`)

Starting from modern Node.js versions, it is recommended to use the promise-based API for better asynchronous flow control.

> [!IMPORTANT]
> **Compatibility Note:**
> - **v14+:** Use `require('node:fs/promises')`.
> - **v10 to v14:** Use `require('fs').promises`.
> - **v8 to v10:** Use `util.promisify` to wrap `fs.open`.

### Using `fs/promises` (Node.js v14+)

```javascript
const fs = require('node:fs/promises');

async function example() {
  let filehandle;
  try {
    filehandle = await fs.open('/Users/joe/test.txt', 'r');
    
    console.log('File Handle FD:', filehandle.fd);
    const content = await filehandle.readFile({ encoding: 'utf8' });
    console.log(content);
  } catch (err) {
    console.error(err);
  } finally {
    if (filehandle) {
      // Always close the file handle!
      await filehandle.close();
    }
  }
}

example();
```

### Using `util.promisify` (Legacy)

If you are on an older version of Node.js or prefer working with existing `fs` methods:

```javascript
const fs = require('node:fs');
const util = require('node:util');

async function example() {
  const open = util.promisify(fs.open);
  const fd = await open('/Users/joe/test.txt', 'r');
  console.log('Promisified FD:', fd);
}

example();
```

## Next Steps
