# The Node.js EventEmitter

In browser-side JavaScript, user interactions are handled through events like mouse clicks and keyboard presses. On the backend, Node.js provides a similar, powerful system using the `events` module.

The core of this module is the `EventEmitter` class, which allows you to create, trigger, and listen for custom events.

---

## Initialization

To use it, you need to require the `events` module and instantiate the `EventEmitter` class.

```javascript
const EventEmitter = require('node:events');
const eventEmitter = new EventEmitter();
```

---

## Core Methods: `on` and `emit`

The `EventEmitter` object primarily exposes two methods that you'll use most of the time:

- **`on(eventName, listener)`**: Used to add a callback function (listener) that will be executed when the specified event is triggered.
- **`emit(eventName, ...args)`**: Used to trigger (emit) an event.

### Basic Example

```javascript
// Define a listener for the 'start' event
eventEmitter.on('start', () => {
  console.log('Started!');
});

// Trigger the 'start' event
eventEmitter.emit('start');
```

---

## Passing Arguments

You can pass data to the event handler by providing additional arguments to the `emit()` method.

### Single Argument
```javascript
eventEmitter.on('start', number => {
  console.log(`Started with code: ${number}`);
});

eventEmitter.emit('start', 23);
```

### Multiple Arguments
```javascript
eventEmitter.on('range', (start, end) => {
  console.log(`Range started from ${start} to ${end}`);
});

eventEmitter.emit('range', 1, 100);
```

---

## Other Useful Methods

The `EventEmitter` class provides several other methods for managing event listeners:

| Method | Description |
| :--- | :--- |
| **`once()`** | Adds a **one-time** listener. After the event is triggered once, the listener is automatically removed. |
| **`off()`** / **`removeListener()`** | Removes a specific listener from an event. |
| **`removeAllListeners()`** | Removes all listeners for a specific event (or all events if no name is provided). |

> [!TIP]
> Memory management is important when working with events. Always remember to use `off()` or `removeListener()` when a listener is no longer needed to prevent memory leaks.
