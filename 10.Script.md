# Running Node.js Scripts & Task Runner

Node.js provides several ways to execute JavaScript from the command line, ranging from basic file execution to a built-in task runner.

---

## 1. Basic Execution

The most common way to run a Node.js program is to use the `node` command followed by the filename:

```bash
node app.js
```

### Using Shebangs
You can make a script self-executing by adding a "shebang" line at the very top. This tells the OS which interpreter to use.

**Recommended Portable Pattern:**
```javascript
#!/usr/bin/env node
// Your JavaScript code here
```

> [!TIP]
> To run a file with a shebang directly (e.g., `./app.js`), you must grant it executable permissions:
> `chmod u+x app.js`

---

## 2. In-line Execution (`--eval`)

To execute a string of JavaScript code directly without a file, use the `-e` or `--eval` flag.

```bash
node -e "console.log(1+2)"
```

> [!CAUTION]
> **Windows/cmd.exe:** Single quotes (`'`) often fail in the standard CMD shell. Always use double quotes (`"`) for the script string on Windows.

---

## 3. Development Mode (`--watch`)

As of Node.js v18.11.0+, you can use the built-in `--watch` flag to automatically restart your application when files are modified.

```bash
node --watch app.js
```

This eliminates the need for third-party tools like `nodemon` for simple development workflows.

---

## 4. Built-in Task Runner (`--run`)

Node.js (v22+) includes a built-in task runner that can execute scripts defined in your `package.json`. It is designed for high performance and simplicity.

### Example `package.json`
```json
{
  "type": "module",
  "scripts": {
    "start": "node app.js",
    "dev": "node --run start -- --watch",
    "test": "node --test"
  }
}
```

### Running Scripts
Use the `--run` flag followed by the script name:
```bash
node --run test
```

### Passing Arguments
You can pass additional arguments to a script using the `--` separator:
```bash
# This passes --watch to the underlying command in the 'dev' script
node --run dev
```

### Environmental Context
The task runner automatically injects specific environment variables:
*   `NODE_RUN_SCRIPT_NAME`: The name of the script being executed (e.g., `test`).
*   `NODE_RUN_PACKAGE_JSON_PATH`: The absolute path to the `package.json` being used.

---

## Intentional Limitations
Unlike `npm run` or `yarn run`, the Node.js task runner is **intentionally lightweight**:
*   No support for `pre` or `post` scripts (e.g., `pretest` will not run automatically).
*   Focuses on performance and low overhead.
*   Ideal for straightforward automation and CI/CD pipelines.
