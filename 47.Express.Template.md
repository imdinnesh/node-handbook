# Using Template Engines with Express

A template engine enables you to use static template files in your application. At runtime, the template engine replaces variables in a template file with actual values, and transforms the template into an HTML file sent to the client. This approach makes it easier to design an HTML page.

The Express application generator uses **Pug** as its default, but it also supports **Handlebars**, **EJS**, and many others.

## Configuring Express for Templates

To render template files, set the following application setting properties in your `app.js`:

1.  **`views`**: The directory where the template files are located.
    *   Example: `app.set('views', './views')`
    *   *Default*: The `views` directory in the application root directory.
2.  **`view engine`**: The template engine to use.
    *   Example for Pug: `app.set('view engine', 'pug')`

> [!IMPORTANT]
> You must install the corresponding template engine npm package. For example, to install Pug:
> ```bash
> $ npm install pug --save
> ```

## How Template Engines Integrate

Express-compliant template engines (like Pug) export a function named `__express(filePath, options, callback)`, which `res.render()` calls to render the template code.

> [!NOTE]
> Some template engines do not follow this convention. The [@ladjs/consolidate](https://github.com/tj/consolidate.js) library addresses this by mapping popular Node.js template engines to the Express-compliant convention, allowing them to work seamlessly.

## Rendering a Template

After the `view engine` is set, you don’t have to manually load the template engine module in your app; Express loads it internally.

### 1. Create a Template
Create a Pug template file named `index.pug` in the `views` directory:

```pug
html
  head
    title= title
  body
    h1= message
```

### 2. Create a Route
Create a route to render the `index.pug` file. If the `view engine` property is set, you can omit the file extension.

```javascript
app.get('/', (req, res) => {
  res.render('index', { title: 'Hey', message: 'Hello there!' })
})
```

When you make a request to the home page, the `index.pug` file will be rendered as HTML.

## Caching Behavior

> [!NOTE]
> The view engine cache does not cache the contents of the template’s output, only the underlying template itself. The view is still re-rendered with every request even when the cache is on.
